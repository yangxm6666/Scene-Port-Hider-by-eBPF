# Build with Android Studio

Android Studio can build this module through NDK CMake, but the BPF object is a
custom host-tool step:

- `hideport_loader`: Android `arm64-v8a` executable built by the NDK toolchain.
- `hideport.bpf.o`: eBPF bytecode built by host `clang -target bpf`.
- `hideport.skel.h`: generated by host `bpftool`.

## 1. Prepare files and tools

Generate `src/vmlinux.h` from the target phone:

```sh
adb shell su -c "bpftool btf dump file /sys/kernel/btf/vmlinux format c" > src/vmlinux.h
```

Build or provide Android arm64 static libraries:

- `libbpf.a`
- `libelf.a`
- `libz.a`

You also need host `clang` with BPF target support and host `bpftool`.

The quickest WSL/Linux path is:

```sh
export ANDROID_NDK=/path/to/android-ndk-r25c
./build_deps_android.sh
```

This creates:

```text
deps/android-arm64/include
deps/android-arm64/lib/libbpf.a
deps/android-arm64/lib/libelf.a
deps/android-arm64/lib/libz.a
```

## 2. Add Gradle/CMake configuration

In your Android app module `build.gradle`:

```groovy
android {
    defaultConfig {
        minSdk 26
        ndk {
            abiFilters "arm64-v8a"
        }
        externalNativeBuild {
            cmake {
                arguments "-DLIBBPF_INCLUDE_DIR=/path/to/libbpf/include",
                        "-DLIBBPF_LIBRARY=/path/to/android-arm64/libbpf.a",
                        "-DLIBELF_LIBRARY=/path/to/android-arm64/libelf.a",
                        "-DZLIB_LIBRARY=/path/to/android-arm64/libz.a",
                        "-DBPF_CLANG=/usr/bin/clang",
                        "-DBPFTOOL=/usr/sbin/bpftool",
                        "-DANDROID_STL=none"
            }
        }
    }

    externalNativeBuild {
        cmake {
            path file("../hideport_module/CMakeLists.txt")
            version "3.22.1"
        }
    }
}
```

For Kotlin DSL, use `arguments += listOf(...)` with the same values.

## 3. Build

In Android Studio, select the `arm64-v8a` variant and run:

- `Build > Make Project`, or
- Gradle task `:app:externalNativeBuildDebug`.

Outputs are copied to:

```text
hideport_module/system/bin/hideport_loader
hideport_module/system/bin/hideport.bpf.o
```

Then package the KernelSU module:

```powershell
.\package.ps1
```

or on Linux/WSL:

```sh
./package.sh
```

## Notes

If loading or attaching fails on the device, check:

```sh
su -c 'cat /data/adb/modules/hideSceneport/hideport.log'
```

The kprobe target symbol name can differ between kernels; the loader logs each
candidate it tries.
